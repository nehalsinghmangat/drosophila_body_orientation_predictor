# drosophila_body_orientation_predictor

Repo for [Mangat et al. (2026)](https://www.biorxiv.org/content/10.64898/2026.03.30.715335v1.full). Predicts *Drosophila* body heading angle from flight trajectory data using a neural network trained on data from [van Breugel et al. (2014)](https://www.sciencedirect.com/science/article/pii/S0960982213015820).

## Installation

### 1. Clone the repo

```bash
git clone https://github.com/nehalsinghmangat/drosophila_body_orientation_predictor.git
cd drosophila_body_orientation_predictor
```

### 2. Download the experimental data

Raw HDF5 data is archived on [Dryad](https://datadryad.org). Download and place the files so the directory tree matches:

```
ExperimentalData/
в”њв”Ђв”Ђ 30cms/
```

### 3. Create and activate a virtual environment

```bash
python3 -m venv .venv
source .venv/bin/activate
```

### 4. Install dependencies

```bash
pip install -r requirements.txt
```

### 5. Add a MOSEK licence (optional)

The convex-optimisation heading correction step in `data_pipeline.ipynb` uses the MOSEK solver via `cvxpy`. Place a valid `mosek.lic` at `~/mosek/mosek.lic`. If you do not have a licence, the simpler naГЇve heading correction is still available and does not require MOSEK. If you are associated with a university and would like to get a free license, please see the installation here: [mosek](https://www.mosek.com/products/academic-licenses/). 

### 6. Launch JupyterLab (or Jupyter Notebook)

```bash
jupyter lab
```

## Workflow

Run the two notebooks in order:

1. **`notebooks/data_pipeline.ipynb`** вЂ” Loads the raw HDF5 files, merges trajectory and body-orientation streams, corrects 180В° heading ambiguities, filters and smooths trajectories, and saves fully preprocessed data to `pipelinedata/06_final/`. Intermediate outputs are written to each numbered subdirectory of `pipelinedata/` for inspection.

2. **`notebooks/model_training.ipynb`** вЂ” Loads the preprocessed data from `pipelinedata/06_final/`, constructs a time-delayвЂ“embedded feature matrix (saved to `pipelinedata/07_time_delay_embedded/`) and a wind-directionвЂ“augmented variant (saved to `pipelinedata/08_wind_augmented/`), runs a hyperparameter grid search, trains the best network, and saves it to `models/model.keras`. Also evaluates predictions on the training dataset and on an external dataset (`pipelinedata/external/`).
